Open Source

Collaborate on SIT Archive

SIT Archive is open source and built by students, for students. Whether you want to add papers, fix a bug, or improve the site — every contribution matters.

How to Contribute — Step by Step

You'll need a GitHub account and Git installed on your computer. No prior open-source experience needed.

1

Fork the Repository

Go to the SIT Archive GitHub repository and click the Fork button in the top-right corner. This creates your own copy of the project under your GitHub account where you can freely make changes.

fork_right Fork sitarchive/sitarchive.github.io
2

Clone Your Fork Locally

Open a terminal and clone your forked repo to your local machine. Replace YOUR_USERNAME with your GitHub username. 

# Clone your fork
git clone https://github.com/YOUR_USERNAME/sitarchive.github.io.git

# Move into the project folder
cd sitarchive.github.io

# Add the original repo as "upstream"
git remote add upstream https://github.com/sitarchive/sitarchive.github.io.git
3

Create a New Branch

Always create a new branch for your changes — never work directly on main. Name your branch something descriptive, like add-cse-sem3-papers or fix-browse-bug. 

# Create and switch to a new branch
git checkout -b your-branch-name

# Example:
git checkout -b add-cse-aiml-sem4-papers
4

Make Your Changes

Make whatever changes you need. Here are the most common ones:

description Adding a Question Paper

Open js/data.js and find the correct location (institute → branch → batch → semester). Add an entry like this:

{ name: "Data Structures", url: "YOUR_GOOGLE_DRIVE_LINK" }

Upload the PDF to Google Drive, set it to "Anyone with the link can view", and paste the share link as the url.

code Fixing a Bug or Improving the Site

Edit the relevant HTML files (index.html, browse.html, etc.) or css/animations.css / js/ files. Open the HTML file in your browser to preview changes before committing.

5

Commit and Push

Save your changes with a clear, descriptive commit message, then push your branch to GitHub. 

# Stage all changed files
git add .

# Commit with a descriptive message
git commit -m "Add CSE AIML Sem 4 end-sem papers (2024-28 batch)"

# Push your branch to your fork on GitHub
git push origin your-branch-name
6

Open a Pull Request

Go to your fork on GitHub. You'll see a banner saying your branch was recently pushed — click "Compare & pull request". Fill in a clear title and description of what you changed and why, then click "Create pull request".

Good PR description example

Title: Add CSE AIML Sem 4 end-sem papers (2024-28)

Added 3 end-semester papers for CSE AIML branch,
2024-28 batch, Semester 4:
- Machine Learning (0707210401)
- Computer Networks (0707210402)
- Operating Systems (0707210403)

PDFs uploaded to Google Drive with public view access.
check

Wait for Review & Merge

The SIT Archive maintainers will review your pull request. They may ask for small changes or clarifications. Once approved, your contribution gets merged into the main site and goes live for all students. We try to review within a few days!

Tips for a Smooth Contribution

check_circle

Keep Google Drive links public. Set share permissions to "Anyone with the link → Viewer" — otherwise the paper won't load for other students.

check_circle

One feature or fix per PR. Smaller, focused pull requests are easier to review and get merged faster.

check_circle

Double-check paper details. Make sure the institute, branch, batch, semester, subject name, and subject code are all correct before submitting.

check_circle

Sync your fork before starting. Run git pull upstream main before creating a new branch so you're working with the latest version.

check_circle

Not comfortable with Git? Use the Submit Papers page to send papers via email and we'll add them for you.

help

Have questions or need help?

Not sure where to start, or running into issues? Head to the Feedback page and drop us a message — we're happy to guide you through your first contribution.
